In this notebook, I have implemented a Collaborative Filtering Recommender System. Instead of building the matrix factorization mechanics completely from scratch, I utilized the Surprise library to efficiently apply the Singular Value Decomposition (SVD) algorithm and manage the underlying user-item interactions. Here is a breakdown of the workflow:

1. Environment Setup and Dependency Management: I managed the computational environment by resolving compatibility conflicts between NumPy and the scikit-surprise library. This required carefully downgrading NumPy to a stable version and compiling the Surprise library directly from its GitHub repository source to ensure seamless execution.
2. Data Loading: I loaded the benchmark MovieLens 100K dataset using the library's built-in loader. This dataset provides a robust environment for testing, containing 100,000 real movie ratings generated by users.
3. Data Preprocessing: I split the data into 75% training and 25% testing sets. By manually partitioning the dataset before training, I ensured the model could be properly evaluated on unseen data, preventing it from simply memorizing historical ratings.
4. Model Selection and Training: I selected the SVD (Singular Value Decomposition) algorithm to act as the core recommendation engine. This algorithm works by decomposing the user-item matrix to discover latent features representing underlying user preferences and movie characteristics. I fitted this model to the 75% training split.
5. Evaluation: I tested the trained model on the unseen 25% holdout set. To measure the accuracy of the predictions, I calculated the Root Mean Square Error (RMSE), achieving a highly competitive score of approximately 0.9326, indicating a low average deviation from actual user ratings.
6. Targeted Prediction: To demonstrate the practical utility of the model, I extracted a specific prediction for an individual user (ID 196) and a specific movie (ID 302). The model predicted a rating of 4.02, which demonstrated remarkable precision when compared against the user's actual historical rating of 4.
